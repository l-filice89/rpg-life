## What Went Well

### Foundation-first sequencing paid off
Story 1.1 created `user_progress` early, which unblocked Story 1.5 Tutorial persistence without reordering epics. The implementation-readiness report flagged this as a risk; the team had already resolved it in 1.1 AC.

### Story files as implementation contracts
Detailed story specs (tasks, brownfield tables, scope boundaries, file paths) reduced wrong-file and wrong-library mistakes. Each story built cleanly on the previous one's File List and Dev Notes.

### Code review loop caught real defects
- **1.1:** Docker API_URL bake, create-x4 scaffold removal, Resend enforcement
- **1.3:** Middleware at wrong path (`apps/web/middleware.ts` vs `src/middleware.ts`) — auth gate never ran
- **1.4:** Hamburger aria-label / aria-controls bugs
- **1.5:** Dead props, concurrent dismiss race on `markSeen`

### Architecture patterns established
- `(auth)/` vs `(app)/` route groups
- RSC-first with minimal client islands
- tRPC `protectedProcedure` + server caller in RSC layouts
- `packages/api/src/routers/` pattern started in 1.5
- Crystal Path tokens + shadcn primitives in `@rpg-life/ui`

### Incremental test coverage
- Smoke tests from 1.1 (scaffold)
- Auth unit tests (17) + tRPC tests from 1.3
- Page-title + tutorial content unit tests from 1.4/1.5
- Tutorial router integration tests with in-memory SQLite (7 cases)

## Challenges and Growth Areas

### Recurring review themes (appeared in 3+ stories)

| Theme | Stories | Impact |
|-------|---------|--------|
| E2E gaps (authenticated flows, magic-link inbox) | 1.3, 1.4, 1.5 | Manual verification required; deferred to Epic 4 |
| `apps/web` unit tests not in root `smoke`/CI | 1.4, 1.5 | Regressions possible until Epic 4 CI |
| Next.js `src/` path conventions | 1.3 | Middleware location caused silent auth bypass |
| RSC layout + tRPC with no error boundary | 1.5 | API down bricks authenticated shell |

### Scaffold adaptation debt
Story 1.1 required significant create-x4 cleanup (removed dashboard, marketing, AI packages, stale routes). Future scaffolds should document a "post-scaffold checklist" upfront.

### Manual verification dependency
Magic link, first-run Tutorial, and sidebar interactions require Resend + live api/web. Epic 4 session-fixture E2E is the highest-value deferred item.

### UX polish discovered late
Header hamburger alignment (centered column vs viewport-left) was caught during manual QA after 1.4/1.5, not in story AC — fixed post-review.

---

## Key Insights

1. **Forward-provision schema early** — `user_progress` in 1.1 prevented a cross-epic blocker for 1.5.
2. **Middleware and layout paths are footguns** — Always verify against Next.js `src/app/` convention; add smoke test that asserts redirect behavior.
3. **Story scope boundaries work** — 1.4 explicitly deferred Tutorial to 1.5; clean handoff via `onTutorialClick` stub.
4. **Code review with fresh context finds different issues** — Blind Hunter caught dead props; Edge Case Hunter caught concurrent dismiss; Acceptance Auditor validated AC coverage.
5. **Deferred-work.md is the debt ledger** — 20 items tracked; Epic 4 should burn down E2E + CI items first.
