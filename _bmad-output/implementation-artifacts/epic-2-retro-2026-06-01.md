## What Went Well

### Story sequencing matched the core loop
The 2.1→2.7 order held: schema before list, list before header polish, CRUD before filters, filters before empty state. Story 2.7 explicitly deferred FR4 empty to avoid blocking 2.2–2.6 — that boundary held cleanly.

### tRPC `tasks` router matured incrementally
Each story added procedures without rewrites: `list` (2.2) → `create` (2.4) → `update`/`delete` (2.5). Repository layer in `packages/db` stayed the single SQL source of truth. Auth tests consistently assert `UNAUTHORIZED` after review patches.

### RSC + client island pattern scaled
- **Server:** `QuestBoard` page fetch, `QuestBoardHeader`, `QuestBoardEmptyFirst`
- **Client:** `QuestBoardFab`, `CreateQuestSheet`, `EditQuestSheet`, `QuestBoardTaskList` + filter context
No `"use client"` creep into data-fetching layers.

### Binding UX copy stayed consistent
EXPERIENCE.md phrases landed exactly: filter labels, empty state ("No quests yet", "Plant a quest to chart your path"), toast nudges. Story specs with copy tables prevented generic productivity voice.

### Test coverage grew with the epic

### Complexity underestimated in edit/delete (2.5)
Story 2.5 had the highest patch count: overdue rules, Focus gating partial implementation, form/sheet state races, duplicate DOM IDs. Server-side Focus bypass was a security-relevant finding caught in review.

### Filter state architecture evolved mid-story (2.6)
Initial TaskList-local state moved to `QuestBoardFilterProvider` in `app-shell.tsx` for nav persistence (AC #8). Review caught File List / task notes drift. Lesson: update story file when architecture shifts during implementation.

### Drizzle migration tooling friction (2.1)
Legacy Postgres snapshot blocked `drizzle-kit generate`. Manual SQL + meta snapshot fix required. Documented in completion notes — future migrations need meta snapshot discipline.

### Epic 1 retro action item not completed
**P1 (middleware redirect smoke)** from Epic 1 retro was not added to root `smoke`. Auth gate still relies on manual verification. Carries to Epic 3/4.

## Key Insights

1. **Schema forward-provision pays off again** — idempotency columns (`completed_at`, `xp_awarded`, `freshness_multiplier`) in 2.1 unblock Epic 3 complete flow without migration rework.
2. **Two empty states must stay distinct** — 2.7 first-time empty vs 3.6 board-clear; story specs explicitly banned conflating copy.
3. **Filter logic belongs in pure functions** — `quest-board-filters.ts` with unit tests (2.6) made review and regression testing straightforward.
4. **Review patches cluster around sheets and mutations** — Epic 3's complete/reward modals should inherit in-flight guards, accessible names, and idempotency patterns from day one.
5. **`deferred-work.md` is now 70+ lines** — debt is visible and categorized; Epic 4 should burn down E2E + CI + error boundaries first.
